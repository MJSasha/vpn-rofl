# VPN ROFL Manager

Простой веб-интерфейс для управления правилами и прокси-группами в конфигурации **Mihomo (Clash)** через SFTP.

## Особенности
- Редактирование правил `rules` (включение/выключение, изменение типа и группы).
- Управление прокси-группами `proxy-groups`.
- Автоматическая перезагрузка конфигурации Mihomo через API.
- Встроенная система самообновления.

## Требования
Для работы приложения необходим доступ к серверу по **SFTP** (SSH), где установлен Mihomo.

## Установка и запуск

### 1. Подготовка
Приложение поддерживает автоматическое обновление напрямую из **GitHub Releases**. Вам не обязательно иметь установленный Go или Git на целевом сервере.

Вы можете просто скачать подходящий бинарный файл со страницы [Releases](https://github.com/MJSasha/vpn-rofl/releases) и запустить его.

### 2. Запуск через Docker (рекомендуется)
Создайте файл `docker-compose.yml`:

```yaml
version: '3.8'
services:
  vpn-rofl:
    image: mjsasha/vpn-rofl:latest # Или соберите локально
    build: .
    ports:
      - "8050:8050"
    env_file: .env
    restart: always
```

### 3. Настройка переменных окружения
Создайте файл `.env` и заполните данные для доступа к вашему SFTP-серверу:

```env
SSH_HOST=127.0.0.1
SSH_PORT=22
SSH_USER=root
SSH_PASS=your_password
CONFIG_PATH=/etc/mihomo/config.yaml
MIHOMO_API=http://127.0.0.1:9090
```

## Обновление приложения
В интерфейсе приложения предусмотрена кнопка **Update**. 
- При нажатии приложение проверяет наличие новой версии в GitHub Releases.
- Скачивает подходящий бинарный файл (`linux-amd64`, `darwin-arm64` и т.д.).
- Автоматически заменяет текущий исполняемый файл и перезапускается.
- Работает как в Docker, так и при обычном запуске.

---

## Для разработчиков: Выпуск новой версии
Проект использует **GoReleaser** и **GitHub Actions** для автоматической сборки релизов.

1. Измените константу `version` в `main.go`.
2. Закоммитьте изменения: `git commit -am "bump version to v1.0.1"`.
3. Создайте и отправьте тег:
   ```bash
   git tag v1.0.1
   git push origin v1.0.1
   ```
4. GitHub Actions автоматически соберет бинарные файлы под все платформы и опубликует их в релизах.
